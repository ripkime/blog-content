## Project Overview

The primary goal of **Project Alpha** was to decouple a legacy monolithic application into independent, loosely coupled services. The monolith was suffering from deployment bottlenecks, where a single bug fix required a full system redeploy.

Key improvements include:

- **Scalability:** Ability to scale individual services like Inventory or Checkout based on demand.
- **Resilience:** Ensuring that a failure in the Notification service doesn't bring down the Payment gateway.
- **Developer Velocity:** Allowing smaller teams to own, build, and deploy specific domains independently.

## Order Processing Logic

A snippet from the order service showing idempotent processing using Redis locks.

```go title="order_handler.go"
func CreateOrder(c *fiber.Ctx) error {
    // 1. Parse Body
    var dto CreateOrderDTO
    if err := c.BodyParser(&dto); err != nil {
        return c.Status(400).JSON(fiber.Map{"error": "Invalid payload"})
    }

    // 2. Check Redis Lock (Idempotency)
    lockKey := fmt.Sprintf("lock:order:%s", dto.OrderID)
    isLocked, _ := redisClient.Get(ctx, lockKey).Result()

    if isLocked == "1" {
        return c.Status(429).JSON(fiber.Map{"msg": "Processing..."})
    }

    // 3. Acquire Lock (TTL 30s)
    redisClient.Set(ctx, lockKey, "1", 30*time.Second)

    // 4. Business Logic ...
    result := orderService.Execute(dto)

    return c.JSON(fiber.Map{"success": true, "data": result})
}
